1. 입력받은 데이터들을 php로 넘겨줍니다.
   이 부분은 백엔드 개발하시는 분들한테 어떤 예제 참고했는지 확인하시면 그 예제에서
   HTML코드는 뭐 수정해야하는지도 나와있을겁니다.
   
   해당 페이지들
   page_dol_new
   page_dol_new_2
   page_dol_match
   page_mat_new
   page_mat_new_2
   page_mat_match

   페이지가 많지만 형식은 2종류라 같은 코드 쓰셔도 될겁니다.

2. 현재 다음 페이지로 넘어가는 부분을 js 조건문을 사용
   적절한 input값이 다 들어와 있어야 php에 데이터를 넘기고 다음 페이지로 넘어갈 수 있게 합니다. 
   그 이외에는 에러메세지를 뜨고 초기화합니다.
   내용물이 잘 들어와있으면 넘기고 아니면 안넘기는 겁니다.
   1.의 페이지들에 같이 적용됩니다.

3. php에서 데이터들을 받아서 뿌립니다.
   레코드들을 받아서 인터넷 게시판 처럼 보여줍니다.
   인터넷 게시판 예제들 참고하시면 될겁니다.
   검색 기능도 가능하면 포함시켜주셨으면 합니다.

   page_dol_match_list : 돌보미가 입력한 아이디 비밀번호가 일치하는 레코드들을 보여줌
   page_mat_list : DB에 있는 돌보미의 매칭 레코드들을 전부 보여줌
   page_mat_match_status : 맡기미가 입력한 아이디 비밀번호가 일치하는 레코드들을 보여줌

4. page_mat_list에서 데이터를 선택해서 page_mat_new로 넘어가는 방식으로 하면 될겁니다.
   게시판 예제에서 게시글 제목에는 견종, 지역, 날짜 정도만 보여주고
   누르면 게시판 내용 나오는 부분에서 매칭 데이터 전부 보여주고
   버튼 누르면 page_mat_new로 넘어가는 방식

5. page_dol_match_list
   page_mat_match_status에서는 게시글 제목 누르면 내용나오고
   매칭되어있으면 매칭되어있는 데이터도 같이 불러와서 내용으로 보여주거나
   링크로 넘어갈 수 있게 하면 될거 같습니다.
   없으면 에러뜨고요
   삭제도 삭제버튼으로 할 수 있게 하면 될겁니다.

   돌보미/맡기미 데이터 항목중에 어떤 매칭과 연결되어있는지 알려주는 항목이 있어야 하는데(레코드 아이디로 저장)
   DB하시는 분한테 물어보시면 됩니다.




스크립트만 구현하시고 버튼 위치같은건 막 두시면 됩니다
